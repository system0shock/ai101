# Навык ревью кода

Проверяй изменение как практичный ревьюер.

## Порядок

1. Найди, какие файлы изменены.
2. Проверь, есть ли подтверждение: тест, вывод команды или точная ссылка на код.
3. Ищи поведенческие ошибки, лишний объем изменений и отсутствие проверки.
4. Не предлагай большой рефакторинг, если задание просит маленькое изменение.

## Формат ответа

- Находки
- Проверки
- Риск
- Минимальное следующее действие
