# 02. Tools

Цель: пройти цикл explore → read → edit → verify и научиться его контролировать.

## Что готово

- Java-проект в `app/` с одним падающим тестом `appliesBulkDiscountFromTenItems`.
- Встроенные tools: read, grep, glob, edit, shell (`mvn test`).
- Образец полного ответа — `developer-track/expected/review-summary.md`.

## A. Попробовать готовое

Запустите тест сами:

```bash
cd developer-track
mvn test
```

Падение — это часть задания.

> В этом задании цель — увидеть работу самих tools (read, grep, edit, shell).
> В проекте уже подключены subagents `code-reviewer` и `test-investigator`
> с близким назначением; чтобы CLI их не вызвал и не «съел» упражнение,
> в промпте явно запретите делегирование. Делегирование — тема задания 06.

Дайте агенту промпт:

```
Не вызывай subagents — сделай всё сам, чтобы было видно, какие tools
ты используешь напрямую.

Запусти mvn test, найди причину падения теста appliesBulkDiscountFromTenItems,
прочитай нужные файлы, предложи минимальную правку, внеси её и повтори тест.
```

Пример хорошего ответа (структура):

```
1. mvn test → падает OrderServiceTest.appliesBulkDiscountFromTenItems
   expected: 900, actual: 1000.
2. read OrderServiceTest.java:11 — тест передаёт ровно 10 товаров по 100 руб.
3. read DiscountPolicy.java:5 — условие if (itemCount > 10).
   На ровно 10 товаров скидка не применяется (граница не включает 10).
4. edit DiscountPolicy.java:5 → if (itemCount >= 10)
5. mvn test → BUILD SUCCESS
```

Полный образец — `expected/review-summary.md`.

Критерий: тест зелёный после минимальной правки одного оператора.

## B. Контроль tools

Задача: составить промпт, который заставит агента работать в два шага —
сначала показать план диагностики и правки, потом, только после подтверждения,
выполнить. Делегирование на subagents запретить.

Критерий готовности: на ваш запрос агент НЕ начинает редактировать код сразу —
сначала выдаёт план и ждёт подтверждения.

Посмотрите в доках по tools (ссылка выше), какие глаголы агент уважает
в инструкциях («не правь», «сначала покажи», «жду подтверждения»).

## Подсказка

Контроль над tools — это про разделение «понял» и «сделал». Без плана агент
может полезть в `OrderService.java`, а не в `DiscountPolicy.java`, и пройдёт
тест хаком (например, изменив сам тест) вместо минимальной правки правила скидки.

### Если застряли

> Подглядывайте сюда, только если самостоятельно не получается.

Промпт:

```
Не вызывай subagents. Не правь код.
Сначала покажи план: какие файлы прочитаешь и какую правку ты предложишь.
На план жду моё подтверждение, потом выполняй.
```

Пример хорошего ответа:

```
План:
1. mvn test — увидеть точное сообщение об ошибке.
2. read OrderServiceTest.java — понять, что передаёт тест.
3. read DiscountPolicy.java — найти граничное условие.
4. Предложить правку (diff).
5. После вашего OK — внести и повторить mvn test.
Подтвердите?
```
