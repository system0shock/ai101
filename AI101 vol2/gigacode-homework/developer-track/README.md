# Developer track

Трек для разработчиков. Рабочая область — маленький Java-проект в `app/`.

## Сценарий

Интернет-магазин внедрил bulk-скидку: при заказе от 10 товаров скидка 10%.
Бизнес-правило описано в `DiscountPolicy.java`, сервис — в `OrderService.java`.
Один тест падает. Ваша задача — найти баг через GigaCode и починить его минимальной правкой.

## Запуск тестов

```bash
cd developer-track
mvn test
```

На Windows без Unix shell:
```powershell
cd developer-track
mvn test
```

Первый прогон покажет падение — это ожидаемо, это часть задания.

## Правила рабочей области

- Редактируйте только файлы в `app/src/`.
- `tasks/` и `expected/` защищены hook'ом — агент не сможет их менять случайно.

## Про примеры в заданиях

Примеры хороших ответов и образец в `expected/review-summary.md` — это
ориентир по форме и сути, не эталон для дословного сравнения. LLM
недетерминирована, ваш ответ будет отличаться от примера и от запуска к запуску.
Финальный объективный критерий в задании 02 — зелёный `mvn test`, а не
дословный текст диагностики. Подробнее — в главном `README.md`.
