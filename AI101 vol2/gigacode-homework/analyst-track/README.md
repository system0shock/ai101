# Analyst track

Трек для аналитиков. Рабочая область — документы в `docs/`.

## Сценарий

Банк готовит запуск онлайн-кредита «Быстрый старт». Четыре документа описывают продукт,
но написаны разными командами и содержат конфликты:

| Файл | Автор | Статус |
|---|---|---|
| `prd.md` | Product | Утверждён |
| `stakeholder-notes.md` | PM | Рабочие заметки со встречи |
| `faq.md` | Marketing | Черновик, не согласован |
| `release-notes.md` | Release manager | Черновик |

Ваша задача — найти противоречия между документами с помощью GigaCode, не меняя источники,
и подготовить материалы для встречи с владельцами.

## Правила рабочей области

- Читайте и анализируйте `docs/` свободно.
- Не меняйте `docs/faq.md` и `docs/release-notes.md` без явного указания в задании.
- `tasks/` и `expected/` защищены hook'ом.
