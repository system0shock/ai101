# 03. Skills

Цель: понять, чем skill отличается от prompt, и научиться его расширять.

## Что готово

- `.gigacode/skills/doc-review/SKILL.md` — общий skill для анализа документов
  и поиска противоречий в `analyst-track`.

## Документация

- Official Qwen Code Skills: https://qwenlm.github.io/qwen-code-docs/en/users/features/skills/
- Локальная справка: `../../.gigacode/skills/qwen-code-helper/references/skills-hooks.md`

## A. Попробовать готовое

Skill срабатывает автоматически, когда запрос подходит под `description`.
Не нужно явно говорить «применяй doc-review» — достаточно сформулировать запрос на анализ.

> Subagent `doc-analyst` имеет схожее назначение и может перехватить запрос.
> Чтобы увидеть работу skill, а не subagent, явно запретите делегирование.

Промпт:

```
Не вызывай subagents — сделай анализ сам, чтобы был виден применённый skill.

Проверь analyst-track/docs/release-notes.md и analyst-track/docs/faq.md
на завышенные публичные обещания относительно analyst-track/docs/prd.md.
```

Пример хорошего ответа:

```
Применяю skill doc-review.

Findings (faq.md):
- «От 1 часа» (faq.md:8) — противоречит prd.md «до 3 рабочих дней».
  Требование юристов нарушено.
- «До 1 000 000 руб.» (faq.md:12) — превышает утверждённый лимит prd.md
  500 000 руб. вдвое.
- «От 9,9%» (faq.md:16) — занижение ставки prd.md «от 14,9%».

Findings (release-notes.md):
- Соответствует PRD по всем параметрам.

Вопрос владельцу: кто согласовал FAQ перед публикацией?
```

Критерий: в начале ответа агент явно упоминает skill (или ведёт ответ по его правилам:
findings → источники → вопросы владельцу).

## B. Минимальная правка

Задача: научить готовый skill `doc-review` явно искать расхождения числовых
лимитов (суммы, проценты, сроки) между документами.

Критерий готовности: повторный запрос из блока A явно выносит расхождения чисел
(1 000 000 vs 500 000, 9,9% vs 14,9%, 1 час vs 3 дня) в отдельный блок findings.

Посмотрите в доках по skills (ссылка выше), как устроен файл `SKILL.md` и где
именно живут правила, по которым работает skill.

## Подсказка

Не пишите skill с нуля и не меняйте `description` — допишите одно правило
в готовый файл. Триггер останется тем же, новый абзац просто расширит
поведение skill.

### Если застряли

> Подглядывайте сюда, только если самостоятельно не получается.

Откройте `.gigacode/skills/doc-review/SKILL.md` и допишите абзац.

Пример результата:

```markdown
---
name: doc-review
description: Используй для анализа продуктовых документов и противоречий в analyst-track.
---

# Document Review

Ищи противоречия, неясные формулировки, невыполнимые обещания и вопросы к владельцам.
Каждый вывод подкрепляй источником.

Особое внимание: числовые лимиты (суммы, сроки, проценты). Если в разных документах
разные числа — это всегда conflict, даже если на словах слово «противоречие» не звучит.
```
