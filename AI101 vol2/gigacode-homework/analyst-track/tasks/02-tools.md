# 02. Tools

Цель: увидеть цикл explore → read → analyze и научиться его контролировать.

## Что готово

- Четыре документа в `analyst-track/docs/` с реальными конфликтами по запуску онлайн-кредита.
- Встроенные tools: read, grep, glob.
- Образец полного ответа — `analyst-track/expected/contradiction-report.md`.

## Документация

- Official Qwen Code Tools: https://qwenlm.github.io/qwen-code-docs/en/tools/index
- Локальная справка: `../../.gigacode/skills/qwen-code-helper/references/commands-tools.md`

## A. Попробовать готовое

> В этом задании цель — увидеть работу самих tools (read, grep, glob).
> В проекте уже подключён subagent `doc-analyst` с тем же назначением;
> чтобы CLI его не вызвал и не «съел» упражнение, в промпте явно запретите
> делегирование. Делегирование — тема задания 06.

Промпт:

```
Не вызывай subagents — сделай всё сам, чтобы было видно, какие tools
ты используешь напрямую.

Найди противоречия между analyst-track/docs/prd.md, stakeholder-notes.md,
faq.md и release-notes.md. По каждому конфликту дай источник (file:line)
и короткую цитату. Документы не меняй.
```

Пример хорошего ответа (фрагмент):

```
1. Аудитория
   - faq.md:4   «Любой клиент банка»
   - prd.md:7   «Только клиенты с зарплатным счётом»
   Конфликт: маркетинг обещает всем, продукт запускается на сегмент.

2. Срок одобрения
   - faq.md:8   «От 1 часа»
   - prd.md:9   «до 3 рабочих дней»
   Конфликт: публичное обещание противоречит SLA и требованию юристов.
```

Полная версия с 4 конфликтами — в `expected/contradiction-report.md`.

Критерий: в выводе минимум 3 конфликта, у каждого два источника с именами файлов.

## B. Контроль tools

Задача: составить промпт, который заставит агента работать в два шага —
сначала показать план, потом выполнить. Делегирование на subagents запретить.

Критерий готовности: на ваш запрос агент НЕ начинает читать файлы сразу —
сначала выдаёт план и ждёт подтверждения.

Посмотрите в доках по tools (ссылка выше), какие глаголы агент уважает
в инструкциях («не запускай», «сначала покажи», «жду подтверждения»).

## Подсказка

Контроль над tools — это не «запретить всё», а разделение «понял» и «сделал».
Без плана агент тратит токены и иногда уходит в боковые ветки (например,
лезет в `tasks/` или предлагает правку до анализа).

### Если застряли

> Подглядывайте сюда, только если самостоятельно не получается.

Промпт:

```
Не вызывай subagents. Не редактируй файлы. Не запускай shell.
Сначала покажи план поиска: какие файлы прочитаешь и в каком порядке.
Жду моё подтверждение, потом выполняй.
```

Пример хорошего ответа:

```
План:
1. Прочитать prd.md — источник истины по ограничениям продукта.
2. Прочитать stakeholder-notes.md — требования юристов и операций.
3. Прочитать faq.md, release-notes.md — публичные формулировки.
4. Сравнить по параметрам: аудитория, срок одобрения, лимит, ставка.
Подтвердите — выполню.
```
